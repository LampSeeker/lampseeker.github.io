## 목적


거미 로봇 만들기 전에, **Unreal에서 Learning Agents가 정상적으로 학습을 돌릴 수 있는지** 먼저 확인한다.


Epic의 Learning Agents는 UE 플러그인으로, **강화학습과 모방학습을 통해 NPC나 물리 기반 캐릭터를 훈련**하는 용도로 제공되고 있어. 공식 학습 자료에는 주행 에이전트 예제, 헤드리스 학습, TensorBoard 연결까지 정리되어 있다.


## 할 일

- Unreal Engine 설치
- 새 C++ 프로젝트 생성
- **Learning Agents 플러그인 활성화**

- 최소한 아래를 확인:
- Agent 생성

- Action 출력
- Reward 계산

- 학습 로그 그래프가 올라가는 것 확인

## 예상 산출물

{{% details title="1단계. “걷지는 않아도 되는” 물리 거미 로봇 만들기" %}}

## 목적


강화학습 전에 먼저 **학습 가능한 로봇 몸체**를 만든다.


초기에는 절대 ARC Raiders처럼 복잡하게 가지 말고,


**훈련이 될 정도로만 단순한 물리 로봇**부터 만들어야 한다.

{{% /details %}}


---


{{% details title="로봇 구조 제안" %}}

### 1차 모델: 4족 로봇


처음부터 8족은 너무 크다.


**4족 → 6족 → 8족**으로 늘리는 게 안전하다.


### 각 다리

- 다리 1개당 관절 2개
- Hip joint: 위아래 또는 전후 회전
- Knee joint: 굽힘
- 총 제어 자유도:
- 4족 × 2관절 = **8개 Action**

### 몸체

- 박스 형태 또는 간단한 캡슐형 바디
- 무게중심 확인 쉬운 단순 메시

{{% /details %}}


---


{{% details title="Unreal에서 구성 방식" %}}

### 추천 방식

- **Physics Asset + Skeletal Mesh**
- 나중에 실제 로봇 모델로 교체하기 좋음
- 관절은
- `Physics Constraint`
- Angular motor / target orientation 제어

처음에는 Blender에서 정교하게 리깅된 로봇을 만들기보다,


**Unreal에서 단순한 프리미티브와 Constraint로 먼저 실험**하는 게 훨씬 낫다.

{{% /details %}}


---


{{% details title="완료 기준" %}}

- 로봇이 중력에 반응함
- 다리 관절을 블루프린트나 C++에서 움직일 수 있음
- 각 다리를 수동 값으로 움직이면 몸체가 흔들리고 넘어질 수 있음
- 바닥과 접촉 판정이 됨

{{% /details %}}


{{% details title="산출물" %}}

- `BP_QuadrupedRobot`
- `C++ JointControllerComponent`
- 관절 디버그 UI
- 슬라이더로 다리 각도 조작

{{% /details %}}


---


{{% details title="2단계. Learning Agents와 로봇 연결" %}}

## 목적


이제 로봇을 **학습 가능한 Agent**로 만든다.


Learning Agents의 핵심 구성은 크게:

- Observation
- Action
- Reward
- Completion / Reset

이고, 공식 API와 튜토리얼도 이 구조를 중심으로 설명한다.

{{% /details %}}


---


{{% details title="2-1. Observation 설계" %}}

처음부터 너무 많이 넣지 말고, **걷기에 필요한 최소 정보**만 준다.


### 1차 Observation

- 몸체의 기울기
- Pitch, Roll
- 몸체 높이
- 몸체의 선속도
- 전방 속도
- 측면 속도
- 몸체의 각속도
- 각 관절의 현재 각도
- 각 관절의 각속도
- 목표 이동 방향 벡터

### 예시 규모

- 바디 상태: 약 10개
- 관절 상태: 8관절 × 2 = 16개
- 목표 방향: 2~3개

총 **30개 전후 observation**으로 시작.

{{% /details %}}


---


{{% details title="2-2. Action 설계" %}}

여기가 매우 중요하다.


### 추천 1안: 관절 목표 각도 출력


각 관절에 대해:

- `1 ~ 1` 범위 action
- 이를 실제 관절 목표 각도로 매핑

장점:

- 학습이 비교적 안정적
- 토크 직접 출력보다 폭주가 덜함

### 추후 확장

- 관절 torque 직접 제어
- PD 제어기와 결합

ARC Raiders를 완전히 흉내 내려면 나중에는 토크 기반 쪽으로 가는 게 더 물리적이지만, **초기 성공 확률은 목표 각도 제어가 훨씬 높다.**

{{% /details %}}


---


{{% details title="2-3. Reward 설계" %}}

첫 보상은 아주 단순해야 한다.


### 기본 보상

- 앞으로 이동하면 +
- 넘어지면 큰 -
- 몸통이 너무 기울면 -
- 제자리에서 과도하게 떨면 -
- 지나친 관절 사용량에 작은 -

### 예시 개념


```text
Reward =
  전방 이동량 보상
+ 목표 방향 정렬 보상
- 넘어짐 패널티
- 몸체 과도한 기울기 패널티
- 에너지 사용 패널티
```

{{% /details %}}


---


{{% details title="2-4. Episode 종료 조건" %}}

- 몸통이 바닥에 닿음
- 몸체 높이가 기준 이하
- 기울기가 너무 심함
- 제한 시간 초과

{{% /details %}}


---


{{% details title="완료 기준" %}}

- 훈련은 정상 실행됨
- 처음에는 버둥거리더라도,
episode 누적 후 **앞으로 가려는 경향**이 나타남
- reward 평균이 완전히 랜덤 상태보다 개선됨

{{% /details %}}


{{% details title="산출물" %}}

{{% details title="3단계. 평지 보행 성공시키기" %}}

## 목적


이 단계의 목표는 하나다.


> **평지에서 목표 방향으로 계속 앞으로 간다.**


뛰거나 멋있게 걷는 건 나중이다.


일단 **쓰러지지 않고 이동**하면 성공이다.

{{% /details %}}


---


{{% details title="세부 마일스톤" %}}

### 3-1. 꿈틀거리며 이동

- 로봇이 바닥에서 우연히 앞으로 밀려감
- 아직 보행처럼 안 보임

### 3-2. 반복 패턴 생성

- 일부 다리가 번갈아 움직임
- 전진성이 생김

### 3-3. 안정 보행

- 일정 시간 이상 쓰러지지 않음
- 방향이 크게 벗어나지 않음

{{% /details %}}


---


{{% details title="여기서 조정할 것들" %}}

- Action scale
- 관절 제한 각도
- 로봇 질량
- 마찰 계수
- Reward 가중치
- Episode 길이
- Reset 포즈

이 단계는 꽤 오래 붙잡게 될 가능성이 높다.


실제로 강화학습 기반 locomotion은 **모델 구조보다 보상과 물리 안정성 튜닝이 성패를 크게 좌우**한다는 점을 감안해야 한다. Embark 역시 물리 기반 보행을 게임 환경에서 쓰기 위해 자체 레이어와 훈련 시스템을 만들어야 했다고 설명했다.

{{% /details %}}


---


{{% details title="완료 기준" %}}

- 평지에서 일정 거리 이상 전진
- 10초 이상 낙상 없이 이동
- 목표 방향 유지
- 학습 중간 모델 저장 및 재생 가능

{{% /details %}}


{{% details title="산출물" %}}

- `QuadrupedWalk_v1`
- 평지 보행 영상
- 보상 추이 그래프
- 실패 케이스 메모

{{% /details %}}

{{% details title="4단계. 지형 적응: ARC 느낌의 시작" %}}

## 목적


ARC Raiders의 핵심은 단순히 걷는 게 아니라,


> **복잡한 지형에서 자세를 유지하며 이동하는 것**


이다.


Embark의 초기 공개 영상도 **복잡한 지형에서 AI 에이전트가 스스로 걷는 모습**을 강조했다.

{{% /details %}}


---


{{% details title="지형 난이도 순서" %}}

### 4-1. 완만한 경사로

- 오르막
- 내리막

### 4-2. 울퉁불퉁한 노이즈 지형

- 높이 차 작은 랜덤 바닥

### 4-3. 낮은 장애물

- 바닥에 놓인 박스
- 짧은 턱

### 4-4. 복합 지형

- 경사 + 턱 + 울퉁불퉁 바닥

{{% /details %}}


---


{{% details title="Observation 확장" %}}

지형 적응을 하려면 주변 바닥 정보를 넣는 편이 유리하다.


### 추가 observation 후보

- 몸체 앞쪽/주변 raycast 높이 샘플
- 각 발 접지 여부
- 발 위치와 몸체 기준 상대 위치

ARC Raiders와 똑같은 내부 방식은 공개되지 않았지만, **지형을 읽을 센서 정보 없이는 험지 적응에 한계가 생기므로**, raycast 기반 지형 샘플링은 재현 프로젝트에서 합리적인 설계다. 이는 공개 자료를 바탕으로 한 구현상 추론이다.

{{% /details %}}


---


{{% details title="완료 기준" %}}

- 경사로를 일정 확률 이상 통과
- 랜덤 울퉁불퉁 지형에서 낙상률 감소
- 평지 전용 정책보다 험지에서 명확히 성능 향상

{{% /details %}}


{{% details title="산출물" %}}

- `QuadrupedTerrain_v1`
- 다양한 지형 데모 맵
- 성공률 비교표

{{% /details %}}

{{% details title="5단계. 거미 로봇으로 확장" %}}

## 목적


여기서부터 진짜 네가 하고 싶은 **“ARC Raiders 거미 로봇 느낌”**으로 간다.

{{% /details %}}


---


{{% details title="5-1. 4족 → 6족 또는 8족" %}}

### 추천

- **6족 먼저**
- 이후 8족

왜냐하면:

- Action 차원이 급격히 증가
- 관절 수가 늘수록 학습이 더 어려워짐
- 그래도 6족부터는 “기계형 거미” 느낌이 확 난다

### 예시

- 6족 × 2관절 = 12 Action
- 8족 × 2관절 = 16 Action

{{% /details %}}


---


{{% details title="5-2. 이동 기능 확장" %}}

- 직진
- 좌회전 / 우회전
- 목표 위치 따라가기
- 정지
- 방향 전환

### Reward도 확장

- 목표 벡터 방향으로 이동
- 목표 각도 정렬
- 정지 명령 시 속도 감소

{{% /details %}}


---


{{% details title="5-3. 무게감 추가" %}}

- 몸체 질량 증가
- 다리 접지 시 충격감
- 저속이지만 강한 보행
- 흔들림 억제

{{% /details %}}


---


{{% details title="완료 기준" %}}

- 6족 또는 8족 로봇이 목표 방향 이동
- 경사 지형 일부 대응
- 서기 / 걷기 / 회전이 분리된 느낌이 남

{{% /details %}}


{{% details title="산출물" %}}

- `SpiderRobot_v1`
- 6족 보행 영상
- 훈련 정책 파일

{{% /details %}}

{{% details title="6단계. ARC Raiders에서 가장 멋진 부분: 회복 행동" %}}

## 목적


이 프로젝트의 “와” 포인트는 여기야.


> **밀렸을 때 균형을 되찾고, 다리 하나가 못 쓰게 되어도 어떻게든 걷는 모습**


이걸 구현하면 단순 RL 실험을 넘어 **ARC Raiders 재현 프로젝트답다**고 말할 수 있다.

{{% /details %}}


---


{{% details title="6-1. 외력 대응 학습" %}}

훈련 중 랜덤하게:

- 옆에서 힘 가하기
- 앞뒤로 밀기
- 순간 충격 주기

### Reward

- 넘어지지 않음
- 기존 방향 회복
- 안정 자세 복구

{{% /details %}}

**좋아. 처음부터 Unreal로 가자.**
