# 2. 로봇 모델을 언리얼 프로젝트에 넣기


## 2.1. 모델 찾기


  모든 걸 시작하려면 우선적으로 움직임이 가능한 거미 모델을 구하는 게 급선무지만, Ark Raiders에 있는 모델을 구하기는 당연히 불가능하니깐, 다른 에셋을 찾아보는중에


[https://www.fab.com/listings/59d748a6-0510-4470-b440-7b3b0d896f72](https://www.fab.com/listings/59d748a6-0510-4470-b440-7b3b0d896f72)


고마우신 분들께서 모든 애니메이션까지 들어가있는 모델을 무료 배포하시는 걸 찾아내서 이걸 열어보기로 했다.


![무려 애니메이션까지! 감사합니다](/notion-assets/3635b029aea68087932aedf8b2be3700.png)


하지만,  지원하는 Unreal 버전이 5.1까지이기 때문에, 직접 해당 파일을 fbx 파일로 받은 후, 직접 프로젝트에 넣도록 하겠다


## 2.2. 모델 언리얼 프로젝트에 임포트


{{% details title="2.2.1. 모델 분석" %}}

- 먼저 폴더 구조를 정리하고…우선적으로 모델 데이터만 넣은 결과

![](/notion-assets/3635b029aea680ce9facce0118ff5d33.png)

- 각각의 항목은

| 항목                           | 정체                 | 지금 프로젝트에서 역할             |
| ---------------------------- | ------------------ | ------------------------ |
| `Bake_Mech_LOD2_albedo`      | **Base Color 텍스처** | 로봇의 기본 색상과 무늬            |
| `Bake_Mech_LOD2_emissive`    | **Emissive 텍스처**   | 스스로 빛나는 부분. 노란 램프 같은 발광부 |
| `Bake_Mech_LOD2_metalness`   | **Metallic 텍스처**   | 어떤 부분이 금속처럼 반사되는지        |
| `Bake_Mech_LOD2_normal_base` | **Normal Map 텍스처** | 표면의 요철감을 가짜로 표현          |
| `Bake_Mech_LOD2_roughness`   | **Roughness 텍스처**  | 표면이 얼마나 거칠고 반사가 퍼지는지     |
| `M_Mech_LOD2`                | **머티리얼 인스턴스**      | 위 텍스처들을 묶어서 실제 로봇 표면 표현  |
| `M_Mech_LOD3`                | **또 다른 머티리얼 인스턴스** | LOD3용 재질로 보임             |
| `SK_ISO_Mech`                | **Skeletal Mesh**  | 로봇의 실제 3D 몸체 + 스킨 정보     |
| `SK_ISO_Mech_PhysicsAsset`   | **Physics Asset**  | 로봇의 물리 충돌체, 관절 제약, 질량 설정 |
| `SK_ISO_Mech_Skeleton`       | **Skeleton**       | 본 계층, 즉 로봇의 뼈대 구조        |

- 으로,  이 중 중요한 것은
1. 스켈레톤 메시(`SK_ISO_Mech`) → 눈에 보이는 로봇 모델
- 로봇의 3D 형상
- 각 버텍스가 어떤 뼈에 붙는지에 대한 Skin Weight
- 사용할 머티리얼 슬롯
- LOD 정보가 있다면 그 연결 정보

로, Skeleton을 기반으로 애니메이션되어 움직이는 메시

1. 스켈레톤 메시 스켈레톤(`SK_ISO_Mech_Skeleton)`
- **로봇의 뼈대 데이터**
- **메시 자체가 아니라 애니메이션을 공유하고 저장하는 기준 구조**
- 같은 Skeleton을 쓰는 애니메이션은 이 모델에 재생할 수 있음
- 이번 경우에는 이후에:
- `A_ISO_Mech_Idle_01`
- `A_ISO_Mech_WalkForward_01`
- `A_ISO_Mech_WalkBackward_01`

같은 애니메이션 FBX를 임포트할 때


**이 **`**SK_ISO_Mech_Skeleton**`**을 지정**해야 함

1. 로봇의 물리 충돌체(`SK_ISO_Mech_PhysicsAsset``**)**`

**강화학습 프로젝트에서 특히 중요한 물리 충돌**

- Skeletal Mesh에 대해:
- 충돌체
- 강체 바디
- 관절 제약
- 질량
- 물리 시뮬레이션

를 정의한다

{{% details title="2.2.2. 모델 스켈레톤 & 애니메이션 임포트" %}}

- 모델이 얼마나 세분화 되어있는지 확인하기 위해 스켈레탈 메쉬를 더블클릭해서 스켈레톤 트리를 확인해보면

![](/notion-assets/3635b029aea680d797d0dd5467745036.png)

- 감사하게도 다리하나가 여러 본으로 세세하게 분리되어 리깅되어있다

(다시 한 번 정말 감사합니다.)2) 애니메이션 확인

- 애니메이션 전용 폴더를 따로 만든 다음, 애니메이션 fbx 파일 하나를 집어넣어보자

![](/notion-assets/3635b029aea680fbb08fe72846cf1048.png)

- 컨텐츠 임포트 창이 뜨는데, 여기서  `스켈레탈 메시` 탭으로 가서

![](/notion-assets/3635b029aea680e591c6e5b5e6732141.png)

1. `애니메이션만 임포트` 체크

![](/notion-assets/3635b029aea680e5aaf5f4e6eb92f78d.png)


이번 FBX는 로봇 본체를 또 가져오려는 게 아니라,


**Idle 동작 데이터만 기존 로봇에 붙이려는 것**이니까 움직이는 데이터만 가져오고,


(Idle 애니메이션 시퀀스)

1. `스켈레톤`에 기존 Skeleton 지정

![](/notion-assets/3635b029aea68099a85bc76e85d652d5.png)


가져오는 애니메이션을 가져온 스켈레톤에 적용하자


```text
A_ISO_Mech_Idle_01.fbx의 움직임 데이터
            ↓
SK_ISO_Mech_Skeleton의 본 구조에 연결
            ↓
SK_ISO_Mech 로봇에서 재생 가능
```
